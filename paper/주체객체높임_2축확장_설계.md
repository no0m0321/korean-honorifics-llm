# 주체·객체높임 2축 확장 — 추천 설계

## 0. 합성 논리 (왜 이 설계인가)

5개 설계의 적대평가 점수는 newtask(16) ≈ factorial(16) > agreement(14) > wrapper(13) ≈ content(13)이다. 단순 1등 채택이 아니라, **두 16점 설계를 골격으로 삼고 나머지 3개의 "살릴 아이디어"를 이식하되, 5개 설계를 가로지르는 4대 치명결함을 모두 제거**하는 방식으로 합성했다.

| 출처 설계 | 가져온 것 | 버린 것 |
|---|---|---|
| **factorial** (골격) | 제3자 내장 조작·2인칭 주어 금지·S축 minimal pair·상대축 6→3·패러 3→1·honor_concord·미러쌍·별모듈 | COPA 원문 개작(매체 고갈) — 자작 미니셋으로 대체 |
| **newtask** (골격) | HSC(완성)·HAJ(판정) 신설 태스크·suppletion>inflection·over-honorification·생성↔판정 이중해리 | 압존·간접높임 미통제 — 사전배제 게이트로 보강 |
| **agreement** | H6(직교 오염에 강건)·H9 과잉존대·"측정축 완전교차/통제축 고정/비핵심 1축만" 셀 원칙·honor_rewrite 정중변환 장르 | '드리다=청자겸양' 측정오염 — 3인칭 논항 한정 카운트로 차단 |
| **wrapper** | 최소침습·캐시/파서 보존·1축 직접 비교·REFERENT 프라이밍 통제셀·"결과 한 줄 전하라"로 전파기회 강제 | C1 조사혼입 — 존칭조사 고정으로 재설계 |
| **content** | "직교를 설계 주장이 아닌 데이터(교호 부재)로 입증" 프레이밍·propagation DV | COPA를 매체로 쓰는 것(높임 가능 역할명사 3.1%) |

**제거한 4대 치명결함:**
1. **DV-출력 불일치** (wrapper/factorial): 분류 단답에서 전파 측정 불가 → 측정의 본진을 **생성형 신설 태스크(HSC/HAJ)**로 옮기고, 부착 태스크에는 "결과를 한 문장으로"로 전파기회 강제.
2. **객체축 frequency/fertility 혼동** (factorial/agreement): O+(드리다)는 저빈도, O−(주다)는 고빈도 → **fertility를 공변량**으로, 그리고 **굴절통제판**(여격조사만 토글) 별도 셀로 분해.
3. **측정 직교 오염** (agreement): 래퍼 '주십시오'의 -시-, 청자겸양 '드리다'가 출력에 누수 → **분류기를 "3인칭 논항이 명시된 절"로 한정 카운트**, 채널1은 지시문을 경어중립으로 고정.
4. **압존법/간접높임 정답오염** (newtask/factorial): NP 절대지위만 토글하면 압존 맥락에서 -시-가 오히려 규범 위반 → **inclusion gate에서 압존·간접높임·무생물주어 사전배제**.

## 1. 추천 설계 — 조작·직교성·실제 프롬프트

### 1.1 세 경어 축의 형태소 슬롯 분리 (직교의 언어학적 근거)

| 축 | 형태소 슬롯 | 높임 대상 | 조작 위치 | 본 설계 처리 |
|---|---|---|---|---|
| 상대높임 (1축, 통제) | 종결어미 EF | 청자 = 모델 | 지시문 래퍼 | 6→**대표 3단계** {해라/해요/하십시오} 고정 |
| 주체높임 (새 축 A) | 선어말 -시- EP + 께서 JKS | 문장 주어 = **제3자** | 태스크 내용 | SUBJ {+/−} minimal pair |
| 객체높임 (새 축 B) | 보충법 어간 VV + 께 JKB | 문장 객체 = **제3자** | 태스크 내용 | OBJ {+/−} + 굴절통제판 |

세 슬롯은 한 어절에 `[어간(객체)] + [-시-(주체)] + [종결어미(상대)]` 순으로 직교 적층된다(`여쭈어보시었다` = 객체O+·주체S+·상대=해라체). 이 3중 적층은 영어(어휘경어 부재)·중국어(굴절 부재)·일본어(단일 굴절 -시- 부재, 객체겸양과의 직교교차 미검증)로 재현 불가 — **이것이 "왜 한국어인가"의 형태론적 핵**이다.

### 1.2 직교성 논증 (4중 보장 + 데이터 입증)

1. **지시대상 분리**: 높임 대상을 항상 **제3자(할아버지/교수님/사장님)**로 고정하고 2인칭 주어를 전면 금지(코드 검수) → 주어가 절대 청자가 아니므로 -시-가 상대높임으로 새지 않는다. 난점 1을 구조적으로 해소.
2. **형태소 위치 분리**: EF/EP/어간이 겹치지 않아 출력에서 세 채널 독립 측정.
3. **측정 직교 보강**(agreement 결함 수정): 전파율을 셀 때 **"내용 명제의 3인칭 논항이 명시된 절"에서만** -시-·보충법을 카운트한다. 래퍼 종결부의 -시-(하십니다)와 청자겸양 보조용언 '드리다'(정리해 드리겠습니다)는 카운트 대상에서 제외. 채널1 신설 태스크의 지시문은 **경어중립 단계로 고정**(해요체 1종, 출력 종결부 -시- 누수 최소화).
4. **데이터로 직교 입증**(content 프레이밍): H10에서 상대높임 단계 × (SUBJ/OBJ) 2원 교호가 비유의함을 사후검정으로 보여 "직교"를 설계 주장이 아닌 **데이터**로 확증.

**fertility 혼동 차단**(factorial 결함 수정): O+ 보충법은 저빈도 어휘라 정확도 하락을 honorification으로 귀인하면 오류. 따라서 (a) 각 보충법쌍의 토큰수·코퍼스빈도를 측정해 **fertility를 glmer 공변량**으로 투입, (b) **굴절통제판 셀**(여격조사 에게→께만 토글, 동사 어간 불변: "할아버지께 책을 줬다")을 별도로 두어 '어휘교체 효과'와 '여격존대 효과'를 해리.

### 1.3 실제 프롬프트 — 조작 대비 최소 4쌍

**쌍 1 — 주체높임 minimal pair (HSC 완성, 채널1, 상대높임=해요체 고정)**

| 셀 | 프롬프트 | 기대 정답 |
|---|---|---|
| SUBJ− | `다음 상황을 한 문장으로 자연스럽게 완성해 주세요.`<br>`상황: 동생이 지금 방에서 (자다).` | 동생이 지금 방에서 잔다. |
| SUBJ+ | `다음 상황을 한 문장으로 자연스럽게 완성해 주세요.`<br>`상황: 할아버지께서 지금 방에서 (자다).` | 할아버지께서 지금 방에서 주무신다. |

조작 = 명사(동생→할아버지) + 주격조사(가→께서) + 보충법주체동사(자다→주무시다). referent를 윗사람으로 고정해 -시- 정당. 명제·난이도 불변.

**쌍 2 — 객체높임 보충법 vs 굴절통제판 (HSC 완성, 상대높임=하십시오체 고정)**

| 셀 | 프롬프트 | 기대 정답 |
|---|---|---|
| OBJ− (평어) | `아래 상황의 빈칸을 채워 한 문장을 완성해 주십시오.`<br>`상황: 민수가 동생에게 책을 (주다).` | 민수가 동생에게 책을 준다. |
| OBJ+ (보충법) | `… 상황: 민수가 할아버지께 책을 (주다).` | 민수가 할아버지께 책을 드린다. |
| OBJ+ (굴절통제판) | `… 상황: 민수가 할아버지께 책을 (주다). [어간 유지]` | 민수가 할아버지께 책을 준다. |

세 번째 셀이 fertility 혼동 분해용 — referent·조사는 존대인데 어간만 평어. OBJ+(보충법)−OBJ+(굴절통제)가 **순수 보충법 어휘 효과**.

**쌍 3 — 경어 적절성 판정 (HAJ, 생성↔판정 이중해리, 상대높임=하십시오체)**

| 셀 | 프롬프트 | 기대 정답 |
|---|---|---|
| 적절(정답 O) | `다음 문장이 경어법에 맞는지 O/X로 판정하고, 틀리면 바르게 고쳐 주십시오.`<br>`문장: 저는 어제 교수님을 뵈었습니다.` | O |
| 위배(정답 X+교정) | `… 문장: 할머니께서 거실에 있다.` | X. 할머니께서 거실에 계신다. |

판정 정확도(haj_judge_correct)와 교정문의 실제 경어 타당성(haj_correction_valid)을 한 태스크에서 동시 측정 → "규칙은 알지만 생성 못 함" 해리(H7).

**쌍 4 — 입력 -시- 전파 미러링 (memo_summ, 최소대립, 채널1)**

| 셀 | 프롬프트 | 측정 |
|---|---|---|
| 입력 -시- 없음 | `아래 메모를 한 문장으로 정리해 주세요.`<br>`메모: 김대리가 3시에 회의실로 온다고 했다.` | 출력 '온다/왔다'(-시- 없음)? |
| 입력 -시- 있음 | `… 메모: 부장님께서 3시에 회의실로 오신다고 하셨다.` | 출력 '오신다고/오셨다'(-시- 전파)? |

**쌍 5 (보너스) — 의뢰자 래퍼 + 전파기회 강제 (NSMC 부착, 채널2, wrapper 살림)**

| 셀 | 프롬프트 |
|---|---|
| REFERENT−hon | `친구가 부탁한 다음 리뷰의 감정을 분류하고, 결과를 그 친구에게 한 문장으로 전해라.`<br>`리뷰: …` |
| REFERENT+hon (프라이밍 통제, 존칭조사 고정·서술어만 -시- off) | `선생님께서 부탁하신 다음 리뷰의 감정을 분류하고, 결과를 선생님께 한 문장으로 전해라.` |

채널2의 핵심 — "결과를 한 문장으로 전하라"가 출력에 의뢰자를 강제 등장시켜 **분류 단답에서도 전파 측정 기회를 확보**(wrapper 치명결함 수정). 프라이밍 통제셀은 wrapper의 C1 조사혼입을 피해 **존칭조사를 양 셀 모두 고정**하고 서술어 -시-만 토글.

## 2. 요인설계 구조와 셀 수 산식 (비대칭 통제)

**셀 원칙(agreement 살림):** 측정 핵심축(SUBJ/OBJ)은 완전교차, 통제축(상대높임)은 대표값 고정, 비핵심 태스크는 1축만.

| 블록 | 태스크 | 상대 | SUBJ | OBJ | 패러 | 아이템 | 모델 | 응답/3모델 |
|---|---|---|---|---|---|---|---|---|
| **A** 핵심 직교 | HSC 완성 | 3 | 2 | 2(+굴절통제 1) | 1 | 100 | 3 | 3×2×3×100×3 = **5,400** |
| **B** 이중해리 | HAJ 판정 | 1(해요체) | 2 | 2 | 1 | 100 | 3 | 1×2×2×100×3 = **1,200** |
| **C** 미러링 | memo_summ | 3 | 2 | — | 1 | 80 | 3 | 3×2×80×3 = **1,440** |
| **D** 부착·비교 | NSMC(채널2) | 3 | 2(REFERENT) | — | 2 | 400 | 3 | 3×2×2×400×3 = **14,400** |
| **소계** | | | | | | | | **22,440** |
| **E** 크기사다리 | HSC 핵심셀 | 1 | 2 | 2 | 1 | 100 | EXAONE 2.4B | 1×2×2×100 = **400** |
| **총계** | | | | | | | | **≈ 22,840** |

**산식 일반형:** `N = |상대| × |SUBJ| × |OBJ| × |패러| × |아이템| × |모델|`

**비대칭 통제 효과:** 만약 통제 없이 6상대 × 3패러 × 2×3(OBJ) × 4태스크였다면 모델당 수만 셀. 상대 6→3, 패러 3→1(A/B/C), OBJ를 핵심 태스크에만, NSMC는 REFERENT 2축만 → **1축 113,400의 약 20%**. M5 Pro 24GB·greedy·Ollama 4bit, 짧은 출력(HSC≈48·HAJ≈64·memo≈64·NSMC≈64 토큰)으로 **모델당 약 7,500응답 → 수~십수 시간, 3모델 합 수일** 완주. 비용 ₩0.

**runner 스키마:** `level` 컬럼을 복합키 `R{1|5|6}_S{+|-}_O{+|-|c}`로 인코딩(스키마 무변경, 캐시·재개 재사용). 아이템 메타(subj±, obj±)는 jsonl 필드.

## 3. 가설 H5 ~ H12 (1축 H1~H4에 이어서)

| 번호 | 가설 | 출처/검정 |
|---|---|---|
| **H5** (정확도 직교) | 2축(SUBJ/OBJ)은 1축(상대높임)과 달리 태스크 **정확도에 거의 영향 없다** — 명령 본동사 형태 불변, fertility 충격 없음. 1축 '공손≠성능'을 이중 보강. | HSC/HAJ acc, glmer |
| **H6** (보충법>굴절) | 객체높임 보충법(어휘교체) 전파율·교란이 주체높임 -시-(굴절) 보다 크다 — fertility 공변량·굴절통제판 통제 후에도. suppletion>inflection. **직교 오염에 강건한 1순위 기둥.** | OBJ_prop vs SUBJ_prop, 셀 내 비교 |
| **H7** (생성↔판정 이중해리) | HAJ 판정 정확도 ≫ HSC 완성 정확도인 모델 존재 = "규칙은 알지만 생성 못 함". 1축 형식회피↔추론손상 해리의 경어판. | haj_judge vs hsc_correct |
| **H8** (모델 비대칭 재현) | 1축 미러링 비대칭(EXAONE 51.3% vs Qwen3 무반응)이 -시-/보충법 전파에서도 재현. 전파율 EXAONE>Qwen3>Llama. | 전파율 × 모델 |
| **H9** (과잉존대 편향) | SUBJ−/OBJ− 평어 입력인데 -시-/보충법 자발 생성(over-honorification) 비율이 한국어특화 모델에서 높다 — 학습된 디폴트 공손. **입력 평어라 청자높임 누수와 구분 쉬워 방어 가능.** | over_hon_rate |
| **H10** (직교 데이터 입증) | 상대높임 단계 × (SUBJ/OBJ) 2원 교호 비유의 → 직교를 데이터로 확증. 유의하면 그것이 더 흥미로운 발견. | glmer 교호항 |
| **H11** (S×O 교호) | SUBJ+/OBJ+ 동시(드리신다) 셀에서 정확도·전파가 단일축 곱보다 낮은 비가산 간섭이 있는가. 2×2 완전요인이라 추정 가능. | 교호항 |
| **H12** (압존 민감도, 부록) | 압존 맥락(주어<청자) 소수 프로브에서 모델이 -시- 누락 규범을 따르는가 vs NP 절대지위에만 반응하는가. | 부록 robustness |

## 4. 새 종속변수 — 경어 일치 전파 측정법

기존 DV(acc_parsed/acc_all·거부율·길이·distinct-n/TTR·ROUGE-L·출력 상대높임 어체) 유지 + 신규 6종. **모두 "내용 명제의 3인칭 논항이 명시된 절"에서만 산출**(측정 직교 보장).

| DV | 정의 | 측정식 | 비고 |
|---|---|---|---|
| **SUBJ_propagation** | 제3자 주어 절 중 -시- 일치 비율 | `P(출력 -시- \| 입력 SUBJ+)` − `P(··· \| SUBJ−)` = Δ | 분모 0(전파기회 없음)은 별도 집계 |
| **OBJ_propagation** | 제3자 객체 절 중 보충법 동사 비율 | `P(보충법 \| OBJ+)` − `P(··· \| OBJ−)` = Δ | 보충법쌍별 집계 |
| **honor_concord** | 한 출력 내 -시-·께서·께·보충법이 상호 모순 없이 일치 | 자질벡터 해밍 유사도 0~1 | 부분모방 진단 |
| **over_honorification** | 입력 평어(−)인데 -시-/보충법 자발 생성 | 이진율 | H9, 디폴트 존대 편향 |
| **under_honorification** | 입력 존대(+)인데 출력 평어로 강등 | 이진율 | 전파 실패 |
| **haj_correction_valid** | HAJ 교정문에 다시 분류기 적용 → 실제 옳은 경어인가 | 이진 | 판정·생성 동시 측정 |

**전파율 보고 형식:** 1축 어체 미러링 지표와 동일 메트릭 가족 `Δ = P(mark|+) − P(mark|−)`로 통일 → 서사 연속성. **측정의 본진은 생성형 태스크(HSC/memo_summ/HAJ교정/NSMC 채널2 "결과 한 줄")**. 분류 단답 셀은 전파 분모 분리 집계해 H5(정확도 직교)와 전파를 깨끗이 분리.

## 5. style_classifier 추가 규칙

기존 `style_classifier.py`(EF 종결어미 = 상대높임)는 **무수정**. 새 모듈 `honor_axes.py`로 분리(기존 정본 회귀위험 0). Kiwi 형태소 태그 기반, κ 생략(이진 수렴 근거).

### 5.1 주체높임 탐지 `detect_subject_hon(text)`

- **-시- EP 탐지:** `tok.tag == 'EP' and tok.form in {'시','으시'}`.
- **보충법 주체동사 사전**(-시- 어간 융합형, EP로 안 떨어지는 부류 보강):
  `_SUBJ_HON_STEMS = {계시, 주무시, 잡수시, 자시, 드시, 편찮으시, 돌아가시}`
- **3인칭 논항 한정:** 같은 절에 제3자 주어 NP(NNG/NNP + 께서/JKS, 또는 래퍼 주입 고정어휘) 존재 시에만 카운트. 래퍼 종결부 -시-와 분리.
- **동음 분기:** 드시다(주체,먹다) vs 드리다(객체,주다) 어간 사전 분기.

### 5.2 객체높임 보충법 탐지 `detect_object_hon(text)`

```
_OBJ_HON  = {드리:주, 여쭙:묻, 여쭈:묻, 모시:데리, 뵙:보, 뵈:보, 말씀드리:말하}
```
- Kiwi 어간(VV/VX) 형태 매칭 + 여격조사 께(JKB) 보조신호.
- **보수적 화이트리스트만 인정**(Kiwi 희소형태 오분석 전례 대응). 우언형(들려주다·갖다드리다)은 별도 정규식 보조 탐지로 커버리지 갭 명시.
- '말씀'(NNG)은 동사 객체높임과 혼선 → '말씀드리/VV'만 인정, 단독 '말씀'은 제외.

### 5.3 일치·과잉 함수

- `honorific_agreement(input_feat, output_text)` → concord 0~1, propagation Δ, error 산출.
- `honor_overgen(text, input_cond)` → 입력 평어인데 탐지 양성이면 1.

### 5.4 자가 테스트(`__main__`)

`"할아버지께 여쭈어보시었다"` → subj=1·obj=1 / `"할아버지에게 물어보았다"` → subj=0·obj=0 / `"동생이 책을 드렸다"` → obj=1(단, 주어 동생은 비존대 → 압존 플래그). **최소 100건 연구자 자기라벨 정확도를 부록에 보고**(자가 테스트 2케이스로는 ACL 리뷰 부족 — agreement 평가 반영).

## 8. 사용자가 실제로 정해야 할 결정 (4개)

> 아래는 책임 공저자인 내가 권고안을 달았으나, **연구 방향을 가르는 분기점이라 당신이 확정해야** 한다.

**결정 1 — 신설 태스크(HSC/HAJ) 포함 여부 vs 부착(NSMC 채널2)만.**
신설 태스크는 "정답=경어 일치"라 측정이 깨끗하고 차별성이 크지만 데이터 자작 비용(블록 A+B 200문항 수일)이 든다. 부착만 가면 1축 비교는 쉽지만 전파 표본이 얇다.
→ **권고: 신설 HSC + 부착 NSMC 병행**(둘 다 16점 설계의 핵). 단 시간이 빠듯하면 HAJ(블록 B)를 부록으로 강등.

**결정 2 — 성능 중심(H5 정확도 직교) vs 일치전파 중심(H6/H9 전파·과잉존대).**
파일럿에서 -시- 천장효과가 나오면 H5는 거의 선험적 null(1축 따름정리)이라 ACL 임팩트가 약하다. 전파·과잉존대는 직교 오염에 강건하고 신선하지만 측정 표본·검정력 리스크.
→ **권고: 전파/과잉존대 중심으로 서사를 짜되 H5를 보강 근거로.** 파일럿 단계 3 게이트에서 최종 확정.

**결정 3 — 상대높임 축소 단계: 대표 3단계(해라/해요/하십시오) vs 더 축소(채널1은 해요체 1종).**
채널1 신설 태스크는 출력 종결부 -시- 누수를 막으려면 경어중립 1단계 고정이 측정에 유리하나, 1축 연속성(미러링 교호 H8)을 보려면 3단계가 필요.
→ **권고: 채널1(HSC/HAJ)은 해요체 1종 고정(측정 직교 우선), 채널2(NSMC)·memo는 3단계 유지(미러링·교호 관찰).** 현재 셀 산식이 이 비대칭을 반영함.

**결정 4 — EXAONE 2.4B 크기사다리 포함 여부.**
1축에서 크기축이 강한 발견(2.4B 하오체 취약)을 줬다. 포함 시 +400응답(거의 무비용)이나 분석·서사 부담 증가.
→ **권고: 포함**(HSC 핵심셀만, 한계비용 미미, "한국어특화 효과의 규모 의존성" 서사 강화).
